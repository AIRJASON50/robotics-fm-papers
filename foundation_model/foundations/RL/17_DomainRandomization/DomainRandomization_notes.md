# Domain Randomization for Transferring Deep Neural Networks from Simulation to the Real World

**Authors**: Josh Tobin, Rachel Fong, Alex Ray, Jonas Schneider, Wojciech Zaremba, Pieter Abbeel  
**Affiliations**: OpenAI, UC Berkeley EECS  
**Year**: 2017 (arXiv:1703.06907)  
**Venue**: IROS 2017

---

## 1. Core Problem

本文解决的核心问题是 **Sim-to-Real Transfer (仿真到真实世界的迁移)** 中的 reality gap (真实性鸿沟)。具体来说:

- 在仿真环境中训练的深度神经网络, 直接部署到真实世界时性能严重下降, 原因在于仿真渲染的图像与真实图像之间存在系统性差异 (域偏移)
- 传统解决方案要么需要大量真实世界标注数据 (昂贵且耗时), 要么尝试提高仿真器的逼真度 (工程代价高且永远无法完美复现真实世界)
- 本文提出一种反直觉的思路: **不追求仿真的逼真性, 而是通过大幅随机化仿真环境参数, 使模型将真实世界视为随机化分布中的又一种变体**, 从而实现零样本迁移

聚焦的具体任务是 object localization (物体定位) -- 检测桌面上几何体的 3D 位置, 这是机器人抓取操作的前置步骤。

## 2. Method Overview

DR (Domain Randomization, 域随机化) 的核心方法流程:

```
仿真器 (MuJoCo/渲染器)
    |
    v
随机化渲染参数 (纹理, 光照, 相机位姿, 干扰物体等)
    |
    v
生成大量非真实感仿真图像 + 自动标注的 ground truth 标签
    |
    v
训练深度检测网络 (modified VGG-16)
    |
    v
直接部署到真实世界 (zero-shot transfer)
```

**关键假设**: 如果仿真中的变化量足够大, 那么真实世界就可以被看作仿真分布中的一个采样点。模型被迫学习所有随机化变体的不变特征, 而这些不变特征恰好对应了物体的本质几何属性。

随机化的参数空间:

| 随机化维度 | 具体操作 |
|-----------|---------|
| 纹理 (Texture) | 桌面/物体/地板/天空/墙壁的所有表面随机分配纹理: 纯色 RGB, 双色渐变, 或棋盘格图案 |
| 光照 (Lighting) | 1-4 个光源, 随机位置/方向/光学特性 |
| 相机位姿 (Camera Pose) | 在初始位姿附近随机平移 (10x5x10 cm) 和旋转 (0.1 rad), FOV 缩放 (5%) |
| 干扰物体 (Distractors) | 桌面上放置 0-10 个随机几何干扰物 |
| 随机噪声 (Noise) | 图像上叠加随机噪声 |

网络架构: 修改版 VGG-16, 输入 224x224 图像, 输出物体质心的 (x, y, z) 世界坐标。使用 L2 loss + Adam optimizer, learning rate 1e-4。

## 3. Key Designs

### 3.1 非真实感纹理随机化策略

这是本文最核心的设计贡献。与追求 photorealism (照片级真实感) 的方向截然相反, 作者**故意使用完全不真实的随机纹理**:

- 纹理类型仅有三种: (a) 随机 RGB 纯色, (b) 两个随机 RGB 之间的渐变, (c) 随机颜色棋盘格
- 场景中所有表面 (桌子/地板/天空/墙壁/物体/干扰物) 各自独立随机采样纹理
- 这种策略看似粗暴, 但实验证明**纹理多样性是最关键的因素** -- 在低数据量下, 纹理随机化比物体位置随机化更重要 (Fig. 5)
- 当使用 1000+ 种独特纹理时性能趋于饱和, 少于 1000 种时性能显著下降

**核心洞察**: 模型被迫忽略颜色和纹理信息 (因为训练中完全不可靠), 转而依赖物体的形状和几何结构进行定位。这种 "逼迫泛化" 的机制是 DR 成功的关键。

### 3.2 干扰物体 (Distractors) 机制

在训练场景中随机添加 0-10 个几何干扰物体, 这些干扰物也来自同一物体集合:

- **对实际部署中遮挡和杂乱环境的鲁棒性至关重要**: 消融实验 (Table II) 表明, 移除干扰物后, 在有遮挡的真实场景中误差从 2.4 cm 暴涨到 7.4 cm
- 干扰物训练使模型学会在杂乱环境中分离目标物体, 即使训练中从未见过真实杂乱场景
- 这一设计后来被广泛采用于机器人视觉训练中

### 3.3 无需相机标定的随机化策略

传统视觉方法严重依赖精确的相机标定 (内参和外参), 本文通过**随机化相机参数**来绕过这一需求:

- 相机位置在 10x5x10 cm 范围内随机偏移
- 视角偏移 0.1 rad, FOV 缩放 5%
- 消融实验表明相机随机化带来稳定的精度提升 (Table II: 无相机随机化时误差显著增加)
- 这使得系统部署时无需精确标定, 仅需大致匹配相机与训练时的粗略设置

## 4. Experiments

### 4.1 实验设置

- **物体**: 8 种几何体 (Cone, Cube, Cylinder, Hexagonal Prism, Pyramid, Rectangular Prism, Tetrahedron, Triangular Prism)
- **训练数据**: 纯仿真渲染图像, 每个模型使用约 20,000 张训练样本
- **测试数据**: 480 张真实 webcam 图像, 物体距相机 70-105 cm
- **评估指标**: 物体中心 3D 坐标的定位误差 (cm)
- **网络**: Modified VGG-16, 用 ImageNet 预训练权重初始化卷积层

### 4.2 主要实验结果

**定位精度 (Table I)**:

| 评估条件 | 平均误差 (cm) |
|---------|-------------|
| 仅物体 (无干扰) | 0.7 - 1.5 |
| 含干扰物体 | 0.6 - 1.8 |
| 含遮挡 | 0.6 - 5.8 |
| 全部场景平均 | ~1.5 |

定位精度约为 1.5 cm, 已足够支撑机器人抓取任务。

**消融实验 (Table II)**:

| 消融条件 | Object Only | Distractors | Occlusions |
|---------|------------|-------------|------------|
| 完整方法 | 1.3 +/- 0.6 | 1.8 +/- 1.7 | 2.4 +/- 3.0 |
| 无噪声 | 1.4 +/- 0.7 | 1.9 +/- 2.0 | 2.4 +/- 2.8 |
| 无相机随机化 | 2.0 +/- 2.1 | 2.4 +/- 2.3 | 2.9 +/- 3.5 |
| 无干扰物训练 | 1.5 +/- 0.6 | 7.2 +/- 4.5 | 7.4 +/- 5.3 |

关键发现:
- **干扰物最关键**: 移除后有遮挡场景误差翻 3 倍
- **相机随机化次关键**: 持续稳定地提升精度
- **噪声影响最小**: 几乎不影响结果, 但有助于训练收敛
- **预训练权重并非必需**: 从随机初始化训练在数据充足时可达到相当精度 (Fig. 4), 但预训练在小数据下更高效

**数据量敏感性 (Fig. 4)**: 约 5000 张仿真图像即可达到较好性能, 50,000 张时基本饱和。

**纹理数量敏感性 (Fig. 5)**: 1000+ 种独特纹理时性能趋于饱和。

### 4.3 机器人抓取实验

- 平台: Fetch Robot
- 使用训练好的检测器定位物体, 配合现成的运动规划软件 [41] 执行抓取
- 在 20 个逐渐增加杂乱度的场景中测试, **38/40 次成功定位并抓取目标物体**, 包括在严重遮挡下
- 进一步用 YCB Dataset [3] 的非均匀纹理物体 (Spam 罐头) 测试, 检测器成功在真实食品干扰物中定位目标, 10 次试验中 9 次成功抓取
- 这是首次完全仅用仿真 RGB 图像训练的深度网络成功迁移到真实机器人抓取任务

## 5. Related Work Analysis

本文涉及三个主要研究方向:

### 5.1 Object Detection & Pose Estimation (物体检测与位姿估计)

| 方法类别 | 代表工作 | 与 DR 的关系 |
|---------|---------|------------|
| 传统多视角方法 | Collet et al. [5,6] | 需要真实标注数据, DR 完全避免了这一需求 |
| 深度学习方法 | 需要大量标注 | DR 通过仿真自动获取无限标注 |
| 3D 模型方法 | Peng et al. [30], Su et al. [40] | 使用 3D 模型渲染训练数据, 但未系统随机化 |

### 5.2 Domain Adaptation (域适应)

| 方法 | 思路 | 对比 DR 的优劣 |
|-----|------|-------------|
| DANN/GAN-based | 学习域不变特征 | 需要目标域 (真实) 数据, DR 不需要 |
| Progressive networks [33] | 预训练 + 微调 | 仍需真实数据微调 |
| Pixel-level transfer | 将仿真图转为真实风格 | 额外引入生成器, 计算开销大 |
| Transfer from 3D models [30] | 随机化背景/光照 | DR 的前身工作, 但仅随机化部分参数 |

**DR 的独特优势**: 完全不需要目标域数据, 不需要额外的适应网络, 且可与 domain adaptation 方法互补。

### 5.3 Sim-to-Real in RL (强化学习中的仿真迁移)

- Sadeghi & Levine [34] 是最接近的工作: 在仿真中训练碰撞回避策略用于四旋翼飞行, 但未验证高精度任务
- 传统 iterative learning control 方法 [1] 用真实数据改进动力学模型, 而非视觉
- DR 方法可与 RL 训练无缝结合 -- 这一方向在后续工作 (如 OpenAI Dactyl) 中被验证

## 6. Limitations & Future Directions

### 6.1 当前局限

| 局限 | 具体描述 |
|------|---------|
| 任务范围有限 | 仅验证了简单几何体的定位, 未扩展到复杂形状或纹理丰富的真实物体 |
| 仅限定位任务 | 未涉及 6-DOF pose estimation (六自由度位姿估计), 仅预测 (x,y,z) 质心 |
| 物体已知形状 | 需要目标物体的 3D mesh, 无法处理未知物体 |
| 精度天花板 | ~1.5 cm 误差对于精密装配等接触操作可能不够 |
| 纹理/形状耦合 | 检测器忽略颜色信息 (因训练时颜色被随机化), 对依赖颜色特征的任务可能有害 |
| 随机化范围需手动设定 | 各参数的随机化范围靠经验确定, 没有自动化方法 |

### 6.2 论文提出的未来方向

- 使用更高分辨率相机提升精度
- 优化模型架构选择
- 引入更多随机化维度: 纹理类型扩展, 多相机视角, 立体视觉, 深度信息
- **结合 domain adaptation 方法** -- 这一方向后来被 SimOpt, ADR 等工作实现
- 与 deep RL 结合, 训练更复杂的操作策略

### 6.3 后续发展 (非论文内容, 基于领域知识补充)

- **自适应域随机化 (ADR, Adaptive Domain Randomization)**: 自动调整随机化范围, 解决了手动设定范围的问题
- **域随机化 + 域适应结合**: 如 RCAN 等工作证明两者互补
- **从视觉扩展到动力学**: 物理参数 (质量/摩擦/阻尼) 的随机化成为主流

## 7. Paper vs Code Discrepancies

**本文没有公开的官方代码仓库。** 论文中提供了一个网页链接 `https://sites.google.com/view/domainrandomization/`, 用于展示实验视频, 但不包含代码。

需要注意的复现细节:

| 细节 | 论文描述 | 复现时的潜在问题 |
|------|---------|---------------|
| 渲染器 | 未明确指定使用的渲染引擎 (文中提到 MuJoCo 用于物理仿真) | 渲染器选择会显著影响生成图像质量 |
| VGG-16 修改细节 | 使用较小的 FC 层 (256, 64), 无 dropout | 原始 VGG-16 FC 层为 4096, 差异巨大, 需注意 |
| 超参数搜索 | 仅搜索了 2 种 learning rate x 3 种 batch size | 搜索空间极小, 可能有更优配置 |
| 纹理生成 | 三种简单纹理模式 | 实际实现细节 (棋盘格频率, 渐变方向等) 未完全说明 |
| 训练样本数 | 每个模型约 20,000 张 | 论文多处提及不同数字, 需以 footnote 4 为准 |
| 光照参数 | "随机光学特性" | 具体的光照模型和参数范围未详细说明 |

## 8. Cross-Paper Comparison

### 8.1 DR vs OpenAI Dactyl (2018, "Learning Dexterous In-Hand Manipulation")

OpenAI Dactyl 是 DR 方法的直接继承和大规模扩展:

| 对比维度 | DR (本文, 2017) | OpenAI Dactyl (2018) |
|---------|----------------|---------------------|
| 任务 | 物体定位 (视觉) | 灵巧手物体操作 (视觉 + 控制) |
| 随机化范围 | 视觉参数 (纹理/光照/相机) | 视觉 + 物理参数 (质量/摩擦/阻尼/执行器延迟等) |
| 策略训练 | 仅监督学习 (回归) | RL (PPO + LSTM) |
| 随机化策略 | 固定随机化范围 | ADR (Automatic Domain Randomization, 自动域随机化), 根据策略性能自动扩展范围 |
| 物体复杂度 | 简单几何体 | Rubik's Cube (复杂非凸几何体) |
| 硬件 | Fetch Robot (单臂) | Shadow Dexterous Hand (24 DOF) |
| 规模 | 单 GPU, 数万张图像 | 数千 CPU workers, 数十亿步仿真 |
| 贡献定位 | 提出 DR 概念并验证可行性 | 将 DR 推向极限, 证明可解决极复杂操作任务 |

**核心关系**: Dactyl 验证了 DR 本文的远见 -- "domain randomization could be an important tool for making [complex RL] policies useful on real robots"。Dactyl 的关键创新 ADR 解决了本文手动设定随机化范围的局限。

### 8.2 DR vs ASAP / Sim-Tool-Real (DR 在实际系统中的应用)

| 对比维度 | DR (本文, 2017) | ASAP-like 实际应用系统 |
|---------|----------------|----------------------|
| DR 的角色 | 核心方法, 端到端使用 | 作为训练 pipeline 的一个组件 |
| 仿真复杂度 | 极简场景 (桌面几何体) | 复杂工业/家居环境 |
| 与真实数据关系 | 完全零真实数据 | 通常混合使用仿真 + 少量真实数据 |
| 随机化粒度 | 全局随机化 (每帧所有参数重采样) | 分层随机化, 不同参数用不同分布和频率 |
| 部署环境 | 实验室受控环境 | 开放世界 / 工厂环境 |
| 工程化程度 | 概念验证 | 生产级系统, 考虑实时性/可靠性 |

**DR 在实际系统中的演进**:

1. **从视觉随机化扩展到物理随机化**: 实际系统同时随机化动力学参数 (关节摩擦, 接触刚度, 执行器模型误差等)
2. **从均匀分布到结构化分布**: 实际系统对不同参数使用不同的先验分布, 而非本文的简单均匀采样
3. **与 System Identification 结合**: 先通过系统辨识缩小参数范围, 再在该范围内做 DR, 提高迁移效率
4. **Curriculum 化**: 从窄范围开始逐步扩大随机化幅度, 避免过度随机化导致训练困难

### 8.3 总结对比表

| 特性 | DR (2017) | Dactyl (2018) | 实际工业应用 |
|------|-----------|---------------|------------|
| 随机化参数数量 | ~10 | ~100+ | 视任务而定 |
| 是否需要真实数据 | 否 | 否 | 通常少量 |
| 随机化范围设定 | 手动 | 自动 (ADR) | 混合 (辨识 + 自动) |
| Sim-to-Real gap 处理 | 仅靠 DR | DR + reward shaping | DR + fine-tuning + adaptation |
| 任务复杂度 | 低 (定位) | 极高 (灵巧操作) | 中-高 |
| 计算成本 | 低 | 极高 | 中-高 |
