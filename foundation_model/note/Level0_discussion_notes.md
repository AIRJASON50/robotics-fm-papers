# Level 0 讨论笔记：表征学习与 Transformer 的核心理解

基于 CS2Robotics_Roadmap Level 0 学习过程中的讨论整理。

---

## 1. 什么是表征学习

对 data/input 进行 mapping，到另一个空间中，使得下游任务更容易解决。原始数据（像素、字符、关节角度）直接用很难做下游任务，需要映射到一个更好的空间。

与**特征工程**的区别: 特征工程是人工设计特征提取规则（如边缘检测滤波器、SIFT 描述子、HOG 特征），表征学习是让网络自动学这些规则。特征工程不是"定义标签"。

### Tokenization 和表征学习的区别

| 概念 | 做什么 | 举例 |
|------|-------|------|
| Tokenization | 把输入切成离散单元（预处理） | "Hello world" -> ["Hello", " world"] |
| Representation Learning | 把每个单元映射成稠密向量（模型学到的能力） | "Hello" -> [0.23, -0.15, ..., 0.87] |

文本 tokenization 是纯切分，不含映射。连续信号（图像、动作）的 tokenization 则不可避免地包含映射（需要决定怎么把连续值变成固定维度向量）。

### 表征的输出不一定是稠密向量

| 表征形式 | 例子 | 用在哪 |
|---------|------|------|
| 稠密向量 | Transformer hidden state | 最常见 |
| 离散 token | VQ-VAE codebook index, RT-2 动作 token | 复用 autoregressive 生成 |
| 概率分布 | VAE 的 (mu, sigma) | 表征不确定性 |
| 结构化 | 图、关键点、粒子 | 保留空间/拓扑结构 |

## 2. 好表征的通用标准

Bengio 2012 提出的先验，至今成立：

| 性质 | 含义 |
|------|------|
| 平滑性 | 输入小变化 -> 表征小变化 |
| 分布式 | 用 N 维组合表达 2^N 种概念 |
| 因素可分离 | 不同语义因素编码在不同维度 |
| 层次性 | 高层概念由低层概念组合 |
| 流形结构 | 高维数据分布在低维流形上 |

### 不同领域侧重不同

标准是通用的，但哪个性质最关键取决于领域：

- LLM：组合泛化最重要
- CV：不变性和等变性最重要
- 机器人：**可执行性 (actionability)** 最重要——表征不仅要理解场景，还要能直接指导连续物理动作

这就是 pi_0 要加独立 action expert 的原因——VLM 表征对"理解"好，但对"精确控制"不够。

**pi_0 双权重的原因** (和 Q/K/V 分开是同一个思路): VLM 的梯度方向是"更好地理解语义"，action expert 的梯度方向是"更精确地生成动作"——两个方向可能冲突。分开后各自优化，不互相拉扯。附带效果: (1) 防止灾难性遗忘; (2) 允许两种表征各自满足不同性质（语义对齐 vs 平滑连续）。

### 数学例子：one-hot vs 分布式

6 个物体（红/蓝 x 方/圆/三角）：

```
one-hot: 6 维，任意两个距离相同，"红方"和"红圆"的共性完全丢失
分布式:  2 维 [颜色, 形状]，共享维度的物体距离近，泛化成为可能
```

## 3. Foundation Model 预训练的目标

预训练 = "尽量什么都保留"。理论指标是互信息 I(x; z)，但高维数据中算不出来，所以用代理目标：

| 代理目标 | 近似什么 |
|---------|---------|
| Next-token prediction (GPT) | 序列的条件互信息 |
| 对比损失 (CLIP) | 跨模态共享互信息的下界 |
| 重建损失 (VAE/MAE) | 直接近似 I(x;z) |

**特征筛选不是预训练的目标——那是 fine-tuning 的事。** 预训练保留所有信息，fine-tuning 调整关注重点（放大需要的维度，压制不需要的）。

### Fine-tuning 是"压制"还是"归零"

取决于方式：全参数 fine-tune 可能归零（灾难性遗忘），LoRA 只是压制（原始权重冻结，能力保留）。

## 4. 多模态输入的处理

不是一个 f，也不是完全独立的多个 f，而是混合的：

```
f = g_shared ∘ [e_img, e_text, e_state, ...]
    ╰共享部分╯   ╰──各模态独立编码器──╯
```

不同模态原始数据结构差异太大（图像 2D 像素、文本离散 token、关节角连续向量），需要各自的编码器先映射到统一维度，再由共享部分处理。

pi_0 的做法：SigLIP (图像 -> 2048 维) + Tokenizer (文本 -> 2048 维) + Linear (状态 -> 1024 维) + MLP (动作 -> 1024 维)，图像/文本走 VLM backbone，状态/动作走 action expert，通过 attention 交互。

**注意**: Transformer 内部也分了两组权重（VLM backbone + action expert），不是完全共享。两组权重各自做线性投影，但在 attention 计算时 Q/K/V 被 concatenate 在一起做联合 attention——所以信息交互发生在 attention 层，参数隔离发生在投影层。

## 5. Transformer 的核心贡献

**不是绑定了某个 loss 或训练范式，而是提供了一种新的信息路由方法。**

### Self-Attention 做什么

对序列中的每个 token，同时计算它和所有其他 token 的相关性分数，然后按分数加权聚合信息。

```
Q_i = x_i @ W_Q  -> "我在找什么信息"
K_i = x_i @ W_K  -> "我能提供什么信息"
V_i = x_i @ W_V  -> "我的实际信息内容" (V 不是相关性分数，是被聚合的信息载体)

Attention(Q,K,V) = softmax(Q @ K^T / sqrt(d_k)) @ V
```

除以 sqrt(d_k) 的原因：Q@K^T 的方差与 d_k 成正比，d_k 大时 softmax 饱和（梯度接近零）。缩放后方差回到 1。

### Multi-Head Attention

一层 attention 内部并行运行多组 QKV 计算（如 8 组），每组叫一个 attention head。每个 head 有独立的 W_Q、W_K、W_V 参数（可学习，训练后稳定），各自偏好不同类型的关注模式（语法/位置/指代等）。初始化时随机无区别，训练中因梯度路径不同而分化出不同功能。

```
实际实现 (GPT-2):
  W_Q: 768x768  (8 个 head 的 W_Q 拼成一个大矩阵)
  W_K: 768x768
  W_V: 768x768

  Q = x @ W_Q -> reshape 成 8 个 96 维 head (768/8=96)
  K, V 同理

  每个 head 各自算 attention -> concat 8 个 96 维输出 -> 768 维 -> @ W_O -> 768 维
```

数学上等价于 8x3=24 个小矩阵，工程上合并成 3 个大矩阵一次算完再切分，GPU 更高效。

**两种 head 的区分**:
- Attention head: Transformer 内部的并行计算单元，每层有多个
- Task head (output head): Transformer 最后接的输出层，把表征转为任务需要的格式（分类/生成/动作）

### Attention 没有自己的 loss

它只是网络内部的计算模块。"该关注谁"是从最终 loss（比如 next-token prediction）通过反向传播间接学到的。没有人标注"sat 必须关注 cat"——模型发现关注 cat 有助于预测后续内容，于是 attention 权重自动收敛到这个模式。

**Q/K/V 为什么要分成三组**: W_Q、W_K、W_V 是三组独立参数，各自有独立梯度。如果只用一个 W 同时生成 Q/K/V，梯度会从三条路径回传到同一组参数，而三条路径的优化方向可能互相矛盾（Q 想编码"我需要位置信息"，K 想编码"我是物体"，V 想编码"坐标(3,2,1)"），导致梯度冲突、收敛困难。分开后每组参数只服务一个功能，不冲突。这和 pi_0 分 VLM backbone + action expert 是同一个思路——不同功能的参数分开，避免梯度冲突。

### 对比 RNN

| | RNN | Transformer |
|--|-----|-------------|
| 信息路由 | 串行，逐步传递 | 并行，直接连接 |
| 建立远距离关联 | O(N) 步，信息衰减 | O(1) 步，无衰减 |
| GPU 利用率 | 低（串行瓶颈） | 高（矩阵乘法） |
| Scale 上限 | ~1B（训练时间瓶颈） | 175B+（完全并行） |

Transformer 能 scale 的核心原因不是"更好的表达能力"（理论上 RNN 也能），而是**把串行问题变成并行问题，GPU 利用率从 1% 提升到 100%**。

## 6. 泛化的数学根源：组合的祝福

Bengio 2012 的核心论点：**分布式表征把维度的诅咒变成组合的祝福。**

### 维度的诅咒（局部方法 / one-hot）

```
10 种颜色 x 10 种形状 = 100 种组合
每种组合是独立"区域"，需要每个至少见一次
20 个属性 x 10 种取值 -> 10^20 个样本（不可能）
```

### 组合的祝福（分布式表征 + 参数共享）

```
表征: z = [z_颜色, z_形状]  两个独立维度
网络: f(z) = w_形状 @ z_形状  （发现只有形状相关）

只需要每种形状见一次 -> ~10 个样本
20 个属性 -> ~200 个样本（线性增长）
```

### 泛化的机制

```
见过: [红, 球] -> 能滚    训练样本
没见过: [蓝, 球] -> ?

因为 [蓝,球] 和 [红,球] 共享形状维度的值
而 w_形状 是跨所有样本共享的参数
所以 w_形状 从 [红,球] 学到的知识自动适用于 [蓝,球]
```

**易错点**: 泛化不是"共享权重"单独带来的。必须是**分布式表征 + 参数共享**两者结合。只有共享权重没有分布式表征（如 one-hot），维度不共享，权重再共享也没法泛化。蓝球也不需要在训练数据中出现来"确认"——泛化是数学上的必然，不是统计上的验证。

### 泛化的三个必要条件

1. **表征是分布式的**：不同因素编码在不同维度
2. **参数跨样本共享**：同一个 w 作用于所有输入
3. **数据的生成结构是可分解的**：现实世界确实由独立因素组合生成

第三个条件最深刻——泛化的前提是现实世界的因果结构确实是组合性的。如果"颜色"真的影响"能不能滚"，分离它们就是错的，泛化就会失败。

---

## 7. Level 0 核心结论

- 表征学习 = 学习数据到有用空间的映射
- 好表征的核心不是满秩，而是"用最少的维度编码最多的独立语义因素，且保持平滑和可区分"
- Transformer 是信息路由创新，不绑定特定 loss
- Foundation model 预训练的好坏理论上由 I(x;z) 度量，实际用代理目标近似
- 泛化来自分布式表征 + 参数共享——维度的诅咒变成组合的祝福
- 好的代理目标 + 大规模数据 + 足够大的模型 -> 好表征的性质自然涌现
